C++11 知识点总结

发表于 2019-02-23 | 分类于 学习笔记

constexpr

2, std::initializer_list
3, 统一初始化 NewClass var{“aaa?”, 3…};
4, 类型推导：auto,为了简化表示 如auto someStrangeCallableType = boost::bind(&SomeFunction, _2, _1, someObject);
5, decltype类型推导，双层括号表引用
int ptrTempA = &tempA;
ecltype(ptrTempA) dclTempB; /解引用，必须初始化，故编译不过
decltype和auto都可以用来推断类型，但是二者有几处明显的差异：
1.auto忽略顶层const，decltype保留顶层const；
2.对引用操作，auto推断出原有类型，decltype推断出引用；
3.对解引用操作，auto推断出原有类型，decltype推断出引用；
4.auto推断时会实际执行，decltype不会执行，只做分析。总之在使用中过程中和const、引用和指针结合时需要特别小心。
6, 仿函数就是写个类，然后重载operator()
7, lambda函数是为了解决临时的谓词函数，语法：
[capture list] (params list) mutable exception-> return type { function body }
8， [] // 沒有定义任何变量。使用未定义变量会导致错误。
[x, &y] // x 以传值方式传入(前提)，y 以传引用方式传入。
[&] // 任何被使用到的外部变量皆隐式地以引用方式加以引用。
[=] // 任何被使用到的外部变量皆隐式地以传值方式加以引用。
[&, x] // x 显示地以传值方式加以引用。其余变量以引用方式加以引用。
[=, &z] // z 显示地以引用方式加以引用。其余变量以传值方式加以引用
9, 仿函数(functor)，是通过重载()运算符模拟函数形为的类
10, explicit 单个参数的构造函数不能进行隐式转换 A a = 5;
11, overlode 重载：
同一类中同名不同参
override 覆盖：
多态，virtual 同名同参
overwrite 隐藏，重写：
父子类，1)基类virtual ,子类同名不同参
2）基类不virtual, 同名同参
调用依赖指针类型
12, 空指针类型nullptr取代NULL
13, enum class：不能与整数比较，类型显式指定，可以前置声明（因为可以确定大小）
14, >> 模板函数中可连写（必须带()以示区分，否则报错）
15, 类模板是用来产生模板类的 class template -> template , template class -> class
15, using 当作 typedef 来处理一般类型和类模板的别名
16, 变长参数模板
17, 新的字符串
18, 用户自定义的字面值
19, 多任务的支持：一、多个线程在一个进程中共存的内存模型；二、为线程之间的交互提供支持
20，default:显式让系统为默认构造函数、析构函数、拷贝构造函数以及拷贝赋值运算符提供函数体
delete:禁止编译器自动产生某些函数
21，void f(int i);
template void f(T) = delete;
除了参数为int之外，都不行
22，long long int==至少64bit
23，允许sizeof操作符作用在类型的数据成员上，无须明确的对象
24，线程相关：std::thread std::thread::join std::mutex std::recursive_mutex std::condition_variable std::condtion_variable_any std::async std::future
25, tuple和erlang中的tuple类似
26，share_ptr weak_ptr
27, 随机数生成：三种算法，n种分布
28，std::ref针对std::bind，std::bind本质是一个函数模板，生成一个函数。而bind采用的是值传递，为了改成引用传递，需要std::ref，面std::cref是包装const引用的值
